yt-dlp Python API (verified pattern from RESEARCH.md):
```python
import yt_dlp
ydl_opts = {"quiet": True, "no_warnings": True, "skip_download": True}
with yt_dlp.YoutubeDL(ydl_opts) as ydl:
    info = ydl.extract_info(url, download=False)
# info["subtitles"] -> dict of lang -> formats
# info["automatic_captions"] -> dict of lang -> formats
# info["id"] -> video ID string
```

CRITICAL yt-dlp pitfalls (from RESEARCH.md):
- subtitleslangs MUST be a list: ["ja"] not "ja" — string silently fails
- download=False means metadata only (for has_japanese_subtitles)
- download=True for download_subtitles
- Use glob.glob() to find actual .srt file (filename varies: .ja.srt, .ja-orig.srt, etc.)
- subtitlesformat="srt" converts VTT to SRT automatically
